# Bebop 中的向量计算支持

## 概述

Bebop 的向量计算子系统为与 Buckyball 集成的机器学习加速器提供高吞吐量数据并行操作。向量计算单元处理多通道向量数据上的逐元素和归约操作，支持各种数据类型和操作模式。

最近对 Bebop 向量加速功能的增强包括扩展的操作覆盖、改进的延迟特性，以及增强的库访问模式以实现有效的内存利用。

## 架构

### 向量通道和数据类型

向量计算单元在 16 个并行通道上运行（可配置）：

- **通道数**: 每个向量单元 16 个通道
- **输入精度**: 8 位元素（支持 int8、uint8）
- **输出精度**: 32 位结果（用于累加的 int32）
- **每周期操作数**: 最多 16 个并行元素操作

这种配置在吞吐量与典型 ML 工作负载的面积/功耗效率之间取得平衡。

### 核心计算模块

#### MulOp（乘法运算符）

`MulOp` 模块实现逐元素乘法，可选择累加：

**接口：**

```
输入：
  - valid: 操作有效信号
  - in1[0..15]: 16 × 8 位操作数 A
  - in2[0..15]: 16 × 8 位操作数 B

输出：
  - valid: 结果有效
  - out[0..15]: 16 × 32 位乘积（A[i] × B[i]）
```

**执行语义：**

- 乘法操作的单周期延迟
- 当 `valid` 断言时结果可用
- 通过就绪信号进行反压（对于连续吞吐量保持高电平）

#### VecComputeTop

`VecComputeTop` 包装器编排跨矩阵块的乘法迭代：

```scala
val start = Input(Bool())       // 启动计算
val iter = Input(UInt(16.W))   // 迭代次数
val op1 = Input(Vec(16, UInt(8.W)))  // 16 个操作数 A
val op2 = Input(Vec(16, UInt(8.W)))  // 16 个操作数 B
val res = Output(Vec(16, UInt(32.W))) // 16 个结果
val valid = Output(Bool())     // 结果有效
val done = Output(Bool())      // 计算完成
```

**执行流程：**

1. 在 `iter` 中使用迭代次数断言 `start`
2. 加载操作数 `op1[0..15]` 和 `op2[0..15]`
3. 当 `MulOp` 每个周期产生结果时保持 `active` 高电平
4. 在 `iter` 迭代后，断言 `done`

### 与 Bebop 加速器的集成

向量计算单元连接到更广泛的 Bebop 内存和控制层次：

- **前端**: 译码向量指令，将操作数路由到计算单元
- **内存域**: 处理矩阵/张量数据预取到本地缓冲区
- **库层次**: 内部双端口 RAM 存储中间和最终结果
- **结果写回**: 累加结果通过 TileLink 返回到主内存

## 向量操作

### 支持的操作

Bebop 实现了一组针对密集线性代数优化的核心向量操作：

| 操作 | 描述 | 延迟 | 吞吐量 |
|------|------|------|--------|
| VMUL | 逐元素乘法（int8 × int8 → int32） | 1 周期 | 16 对/周期 |
| VMAC | 乘法-累加链 | 1 周期 | 16 对/周期 |
| VREDSUM | 向量归约求和 | 4–6 周期 | 1 结果/操作 |
| VREDMAX | 向量归约最大值 | 4–6 周期 | 1 结果/操作 |
| VPADD | 并行加法（int32 + int32 → int32） | 1 周期 | 16 对/周期 |
| VSHUFFLE | 通道内混洗用于数据重组 | 2 周期 | 16 元素/周期 |

### 数据流示例：64×64 矩阵乘法

矩阵乘法分解为平铺操作：

```
给定 A[64×64] 和 B[64×64]：
1. 前端加载平铺 A[0..15, 0..15] 和 B[0..15, 0..15]
2. VecComputeTop 执行 16 次迭代：
   - 周期 0：乘法列 0
   - 周期 1：乘法列 1
   - ...
   - 周期 15：乘法列 15
3. 内存中的累加器集成部分结果
4. 重复剩余平铺直到计算完整结果
```

## 内存交互

### 操作数供应

向量操作从本地双端口 RAM（库层次）读取操作数：

- **端口 A（读）**: 当前迭代的操作数
- **端口 B（读）**: 下一次迭代的预取
- **写端口**: 将结果写回到内存

这种双访问支持流水线化的操作调度。

### 结果累加

中间结果在指定的内存区域累加：

- **缓冲区位置**: 对齐到 64 字节边界以实现高效的缓存行传输
- **原子性**: 没有原子操作；软件协调多指令累加
- **一致性**: 写回完成后，Buckyball 核可见结果

### 带宽优化

突发内存操作隐藏延迟：

```
预取模式：
- 当迭代 N 计算时，DMA 加载迭代 N+1 的数据
- 每个 16 通道结果（64 字节）在单个 TileLink 突发中写入内存
```

## 性能特性

### 吞吐量

对于密集矩阵操作：

- **持续吞吐量**: 每周期 16 个乘法-累加操作
- **系统吞吐量**: 受内存带宽限制（通常 8–16 Gbps）
- **利用率**: 典型情况下对于良好平铺的内核 70–85%

### 延迟

每操作延迟（单个矩阵乘法）：

- **乘法**: 1 周期
- **归约**: 4–6 周期（基于树的并行归约）
- **内存往返**: 20–50 周期（取决于缓存状态）

### 跨核扩展

当 Buckyball 运行多线程工作负载时：

- 向量计算与 CPU 核共享 TileLink 总线
- 争用增加内存延迟（每个核预期 1.2–1.5× 减速）
- 调度工具（`GlobalScheduler`）可在内存密集阶段优先考虑 Bebop 操作

## 编程接口

### RoCC 指令格式

向量操作在 RoCC 自定义指令中编码：

```
指令: funct=BEBOP_VMUL, rs1=A_addr, rs2=B_addr, rd=dest_reg
```

RoCC 接口通过 Spike-Verilator 协同模拟路由操作数并接收结果。

### 软件栈

典型 ML 框架用法：

```python
# PyTorch/TensorFlow 风格用法（编译为 RoCC）
C = A @ B  # 矩阵乘法分派到 Bebop
# RTL 执行 VecComputeTop 迭代
# 结果写回到 C 缓冲区
```

### 手动调优

高级用户可以控制：

- 平铺大小和迭代次数
- 用于操作数放置的库选择
- 相对于计算的预取时序

## 调试和分析

### 波形检查

启用向量操作的详细追踪：

```bash
bbdev verilator --run \
  '--batch --vcd veccomp.vcd \
    --binary ctest_bebop_vec_matmul'
```

在波形查看器中，检查：
- `VecComputeTop.io.start`（操作触发）
- `VecComputeTop.io.valid`（结果就绪）
- `MulOp.io.in/out`（操作数/结果总线）

### 执行追踪

启用详细输出以查看操作分派：

```bash
bbdev verilator --run \
  '--batch --verbose \
    --binary ctest_bebop_vec_matmul' 2>&1 | grep "VMUL\|VREDSUM"
```

输出包括：
- 指令 funct 代码
- 操作数值和地址
- 结果延迟和正确性

### 库访问模式

在向量操作后使用 `bankDigestPeek` 验证内部状态：

```scala
// 在协同模拟框架中
val expectedDigest = sha256(A) ^ sha256(B)  // 黄金模型
assert(bankDigest === expectedDigest, "Bank state mismatch")
```

## 故障排除

### 结果不正确

**症状**: 输出与参考值不同（CPU matmul）

**诊断步骤：**
1. 验证操作数数据类型匹配（8 位输入、32 位输出）
2. 检查通道计数配置（必须为 16）
3. 检查内存中的累积部分结果
4. 在出现分歧的周期比较波形

**常见原因：**
- 配置中的通道计数不匹配
- 操作数对齐不正确（必须为向量对齐）
- 累加中的整数溢出（检查缩放）

### 性能低于预期

**症状**: 操作耗时比理论延迟长

**分析：**
1. 分析内存带宽：运行隔离乘法的微基准
2. 检查库冲突：检查波形中的访问模式
3. 验证预取时序：确保下一次迭代数据在需要前到达
4. 监控总线争用：检查 TileLink 忙周期

**优化：**
- 增加每个 RoCC 调用的迭代次数（摊销分派开销）
- 将平铺对齐到库边界以避免冲突
- 在 CPU 核停滞时调度 Bebop 操作

## 集成示例

### 矩阵转置

要计算转置，重组平铺并使用混洗操作：

```
输入：A[4×4]（平铺）
步骤 1：加载 A[0][0..3]，B = A^T（通过 VecComputeTop 混洗）
步骤 2：预取 A[1][0..3]
结果：输出缓冲区中的转置数据
```

### 归约操作

对 16 通道向量求和归约：

```
输入：vec[0..15]
树归约：
  级别 1：vec[0]+vec[1]、vec[2]+vec[3]、...、（8 个结果）
  级别 2：sum[0]+sum[1]、sum[2]+sum[3]、...、（4 个结果）
  ...
  最终：包含所有 16 个元素之和的单个累加器
```
